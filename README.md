## 📚 Descripción del Curso

El curso de Análisis y Desarrollo de Software está orientado a formar profesionales capaces de:
- Comprender los fundamentos del desarrollo de software y el ciclo de vida de los sistemas.
- Analizar requerimientos funcionales y no funcionales.
- Diseñar arquitecturas de software escalables y mantenibles.
- Implementar soluciones utilizando lenguajes de programación, frameworks y bases de datos.
- Aplicar metodologías ágiles (Scrum, Kanban) y control de versiones con Git.
- Realizar pruebas unitarias, de integración y aseguramiento de la calidad.
- Documentar procesos y productos de software.

## ⚙️ Instalación y Configuración

Para clonar y trabajar con este repositorio:

```bash
git clone https://github.com/SantanderAcuna/clase_github_sena.git
cd clase_github_sena
```

Dependiendo del proyecto específico, es posible que necesites instalar:

- **Java JDK 11+** (para proyectos en Java)
- **Python 3.8+** (para proyectos en Python)
- **Node.js y npm** (para proyectos JavaScript)
- **MySQL / PostgreSQL** (para bases de datos)
- **Visual Studio Code**, **IntelliJ IDEA** u otro editor de tu preferencia

Consulta la carpeta correspondiente para ver los requisitos particulares de cada actividad.

## 🚀 Uso

Cada carpeta contiene su propio README con instrucciones detalladas. En general:

- Para ejecutar ejemplos de código, accede a la subcarpeta respectiva y sigue las indicaciones.
- Los proyectos integradores se encuentran en `07_proyecto_final/` y contienen sus propios pasos de ejecución.
- Utiliza los scripts de base de datos en `05_bases_datos/` para crear y poblar las tablas necesarias.

## 🤝 Contribución

Este repositorio es principalmente para uso académico, pero si deseas contribuir:

1. Haz un fork del proyecto.
2. Crea una rama para tu feature (`git checkout -b feature/nueva-mejora`).
3. Realiza tus cambios y haz commit (`git commit -m 'Agrega nueva funcionalidad'`).
4. Sube los cambios (`git push origin feature/nueva-mejora`).
5. Abre un Pull Request.

Todas las contribuciones deben seguir las buenas prácticas de código y estar documentadas.
